# Contributing

Thanks for helping improve `codex-oss`.

## Development

```sh
npm install
npm run check
```

## Change Guidelines

- Keep changes small and maintainer-focused.
- Add or update tests for behavior changes.
- Prefer plain functions over new abstractions.
- Fail fast with clear errors when command input is invalid.
- Do not add runtime dependencies without a concrete maintainer workflow need.

## Pull Requests

Each pull request should explain:

- what maintainer workflow it improves;
- the smallest user-visible behavior change;
- how it was tested.
